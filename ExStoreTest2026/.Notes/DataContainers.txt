> data storage (revised)
	* assume that there is nothing that can be used as a
		unique identifier for the model and that any
		unique information for the model could change
			e.g. name could change, file create date / time could change
				for example when a local model is moved to the cloud
	* validation that the correct DS is found is comparison of the 
		Document associated witht the DS and the document associated with
		the current invocation of the add-in	
	* these are unique per model
	* there is two or more per model
	* model code is just to make the DS names unique per model
	* naming
		> {app_name}_DS_0000_{Model_Id_code}_{version}  [first / prime DS]
		> e.g. CsCells_DS_0000_250101_160101_v1_00  {for example} {contains the workbook data}
		> e.g. CsCells_Sht_AAAA_250101_160101_v1_00 {first of one or more sheet DS objects}
	* {ID} sequence AAAA, AAAB, AAAC ... AAAZ, AAA0 ... AAA9, AABA ... etc. 
		* note that the last number is Z999 (first digit is never a number) so that
			the {ID} of 0000 for the prime DS is unique
