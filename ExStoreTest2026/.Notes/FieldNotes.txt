descriptions of the schema and the fields in the schema


Fields, etc.
* defines the fields of a schema

type 1 data list (one schema for all?)

> (voided) primary schema (name: CsCells_Primary_{model ID code}_20250101A)
        : name suffix is date + revision letter
    * purpose: used to validate the system.  that is, make sure 
        that the original model is being used.
    * process auto run (if / when activated) (on db open event)
        : uses a array list of "rows" to process
    * model ID code: random 6 character code. 
        : specific for this model. used in the names of all schemas
    * components
        > [AD], [SD]
            + [MD]


> sheet schema (name: CsCells_Sheet_{model ID code}_20250101A)
> (voided) sub-schema (rows) (name: CsCells_Row_{model ID code}_20250101A)
        : name suffix is date + revision letter
    * components
        > [AD], [SD]
            + [ED], [OD], array list { [RD] }




* schema components

    * [AD] admin data
        * description
        * created date
        * created by
        * deleted (as in deleted but not purged from memory)
        * modified date
        * modified by
        * model ID code
        * last sheet DS ID

    * [SD] schema data
        * schema version
        * GUID

    * [MD] model data
        * name of model
        * (void) model (file) creation date+time


    * [AR] auto run data
        * array list of "rows" to process

    * [ED] excel data
        * excel file path
        * sheet name
            
    * [OD] operation data
        * active / inactive
        * sequence code
        * update rule: on demand / on db open (if possible) but only if given permission
        * skip updating

    * [RD] revit data
        * linked family name
        or
        * linked family name + type name

copy / duplicate
y - copy the original value
u - update with new value
i - ignore, skip (applies to fixed values)

can edit
1 = basic editor
2 = advanced editor
D = debug editor
Y1, 2, D - viewing by 1+ & edit allowed by 1+, 2+, or D
N1, 2,   - viewing allowed by 1+, or 2+ - editing allowed by D

edit clean / dirty
ig - does not change, cannot be clean or dirty
cl - can change, can be clean or dirty


copy
A - always - as is
N - never - always provide a new value
1 - copy as is
0 - provide a new value

7 - copy always
0 - copy never
2 - copy when type 2
3 - copy when type 1 or 2
5 - copy when type 1 or 4

current copy types
A  type 1 - as a template - allows a new ds to be based on a prior ds
B  type 2 - duplicate to update the version
C  type 4 - duplicate to be used in a new model

examples
copy type 4
copy all
except update these
7 (1+2+4), 6 (2+4), 5 (1+4), 4 (4)

copy type 1
copy 
except update these
7 (1+2+4), 5 (1+4), 3 (1+2)
