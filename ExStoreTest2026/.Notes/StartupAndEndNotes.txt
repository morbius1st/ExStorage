ExStorMgr will be the keeper of the modification status

* create a single control panel
	> btn: save workbook
	> btn: reset workbook (does this also reset sheets?)
	> btn: save sheet(s)
	> btn: reset sheet(s)
	> btn: remove system
	> sheets list is independant
	> family and type list is independant


Status to be reported

* needs to be restarted
* ExStorData / Sheets List is modified
* Workbook is modified
* Sheet (1+) is modified
*	which includes the family & type list being modified
* ExStorMgr - remove ExStor from model

consequences / procedures

* system status
	> enum
	> has changes
	> no changes
	> needs restart

* workbook is modified
	> only if sys status <= has changes (else = disabled)
	> cannot leave system until changes resolved
	> commit changes
	> vacate changes

* sheet is modified
	> only if sys status <= has changes (else = disabled)
	> cannot leave system until changes resolved
	> includes family & type list
	> commit changes
	> vacate changes

* family list is modified
	> part of sheet is modified - cannot leave sheet until changes are resolved
	> commit changes
	> vacate changes

* sheets list
	> can only modify when system status != needs restart
	> adding a sheet = has changes (when sys status != needs restart) (else disabled)
	> deleting a sheet = needs restart (but not has changes) (when sys status = no changes) (else disabled)
